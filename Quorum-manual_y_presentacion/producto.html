Ficha del software QUORUM

Sistema web para digitalizar el control de asistencia de aprendices del SENA — Centro de Procesos Industriales y Construcción (CPIC). Descripción alineada al README y al código del repositorio.

Instalación y ejecución Manual completo

Metadatos del proyecto

Nombre QUORUM
Cliente / contexto SENA — CPIC (según documentación del proyecto)
Idioma UI Español (Colombia)
Repositorio Aplicación dividida en quorum-frontend y quorum-backend

Stack tecnológico (implementado)

  • Frontend: Next.js 14 (App Router), TypeScript estricto, Tailwind CSS, Lucide React, SweetAlert2, Axios.
  • Backend: Laravel 12, PHP 8.2.
  • Base de datos: MySQL 8 (entorno típico XAMPP).
  • API: REST JSON; autenticación SPA con Laravel Sanctum (cookies stateful).
  • Reportes: Excel mediante plantilla storage/app/plantilla_asistencia.xlsx (PhpSpreadsheet en backend).

Arquitectura lógica

┌─────────────────┐ HTTPS/JSON ┌──────────────────┐ │ Navegador │ ◄─────────────────► │ Laravel API │ │ Next.js SPA │ Sanctum cookies │ PHP + MySQL │ └─────────────────┘ └────────┬─────────┘ │ ▼ ┌───────────────┐ │ MySQL 8 │ └───────────────┘

El frontend consume la API bajo el prefijo configurado (p. ej. NEXT_PUBLIC_API_URL=http://localhost:8000). Las reglas de negocio y permisos se aplican en el servidor (policies, middleware, incluido EnsureTotpSessionOk para sesiones staff con 2FA completado).

Roles de usuario

Administrador

Gestión de usuarios, fichas, centros, configuración, festivos, auditoría y catálogos según API.

Coordinador académico

Consulta y seguimiento ampliado vía rutas /api/coordinador/* y vistas asociadas.

Instructor / Gestor de grupo

Toma de asistencia, historial de fichas asignadas, exportación Excel cuando corresponda.

Aprendiz

Solo lectura de su propio historial; login con correo y documento (sin contraseña).

API REST (grupos)

Definición en quorum-backend/routes/api.php:

  • GET /api/ping — comprobación de vida.
  • /api/auth/* — login, logout, me, 2FA, recuperación y reset.
  • Tras autenticación y TOTP de sesión: dashboard, usuarios, fichas, asistencia, reportes Excel, coordinador, configuración, festivos, historial actividad, centros admin, auditoría.
  • GET /api/mi-historial — aprendiz autenticado.
  • PATCH /api/perfil/contrasena — cambio de contraseña en perfil.

Diferenciadores frente a Excel suelto

  • Una sola fuente de verdad en MySQL con políticas por rol.
  • Trazabilidad de sesiones y registros de asistencia; respaldos y auditoría según tablas del sistema.
  • Salida a plantilla Excel CPIC homogénea para cierre de periodo.
  • Flujo de aprendiz aislado: solo ve sus propios datos.
